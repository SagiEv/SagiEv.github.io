// Overview

Full-stack task management application built with a strict n-tier architecture — presentation, business logic, and data access layers are fully separated with no cross-layer dependencies.

Developed collaboratively using structured GitHub workflows with PR reviews, branch protection rules, and issue tracking — simulating a professional engineering team process.

// Key Features
  • Kanban board with column-based task lifecycle (Backlog → In Progress → Done)
  • Board, column, and task CRUD with validation at the business logic layer
  • Clean n-tier separation enforced at the assembly level
  • SQLite persistence via a dedicated data access layer with repository pattern
  • Team PR review process with protected main branch and CI checks
// Challenges & Solutions
CHALLENGE
Enforcing strict n-tier boundaries — early versions had UI code reaching directly into data layer
SOLUTION
Refactored into separate C# projects (assemblies) — compiler enforces layer boundaries via project references
CHALLENGE
Coordinating concurrent feature development across team members without breaking main
SOLUTION
Adopted feature-branch Git workflow with required PR reviews and automated build checks before merge
