Set Game Engine

View on GitHub

// Overview

Concurrent real-time implementation of the card game SET, where multiple threads and human players compete simultaneously in a shared game state. The core challenge was ensuring correctness under heavy parallel execution.

Built with advanced Java synchronization mechanisms — locks, condition variables, and atomic operations — to prevent race conditions and guarantee fair, deterministic game outcomes regardless of thread scheduling.

// Key Features
  • Multi-player concurrent execution with N simultaneous player threads
  • Lock-free claim validation using atomic compare-and-swap operations
  • Dealer thread managing the shared deck and scoring in sync with players
  • Configurable bots with adjustable reaction-time simulation
  • Comprehensive JUnit test suite covering concurrent edge cases
// Challenges & Solutions
CHALLENGE
Two players claiming the same card simultaneously could corrupt the deck state
SOLUTION
Atomic token system: each card slot has an owner token updated via CAS — only one claim succeeds, others retry
CHALLENGE
Dealer must wait for players to finish claiming before reshuffling, without busy-waiting
SOLUTION
Used a CountDownLatch per round — players decrement on claim completion, dealer blocks on await()
